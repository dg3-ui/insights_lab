## Form Lessons (what to imitate)

1. **Dek = the questions** ("What happened, and was the local battery… able to capitalize?") — the piece promises an investigation and a verdict on a *named* asset.
2. **The drill trajectory**: when-and-where (system level) → market mechanics (why prices could do that) → the asset section (did it capture the moment?). This is Bucket 2 feeding Bucket 1 in a single artifact.
3. **The verdict is honest, not heroic**: "It did ok" + degradation context + "hard to predict all-time-highs." An asset-level read that resists both hype and mush — the bottom-up altitude we want in Phase 5.
4. **Charts deep-link to the live platform** with parameters preserved; API queries shown in code blocks. The blog *is* a product demonstration without saying so. **Our version: entity mentions and charts link to infrasure.ai pages where they exist.**
5. **Reader-respecting asides** ("If you have a map of ERCOT substations in your head…") — assumes intelligence, explains anyway.
6. **CEII/data-limit caveats named explicitly** — what cannot be known is said, not papered over.
