## Form Lessons (what to imitate)

1. **Title = concept + consequence** ("…When We Throw Away Clean Energy") — names the phenomenon and what's at stake in one breath, no clickbait.
2. **The explainer ladder**: intro poses the live question → primer defines the mechanism (LMP) → then THREE regional case sections, each a different *cause* of the same phenomenon (demand-driven CA · congestion TX · inflexibility SPP) → technical deep-dive → conclusion reframes as a standing condition to manage, not a problem to eliminate.
3. **One region = one cause = one section.** The fan-out is organized by mechanism, not geography for its own sake — exactly our exposure fan-out shape.
4. **A chart per analytical step** (~7), each placed at the statistic it carries; captions name the data's source and intent.
5. **Caveat craft**: footnote admits the data's imperfection ("does not report curtailment directly"); a dated *correction disclaimer* was added post-publication — public errata as credibility.
6. **Closing job**: zoom out from mechanics to what-it-means (resource-mix choices), no triumphant summary.

## Claims discipline

All factual assertions in the source (curtailment volumes, regional behaviors, LMP mechanics as described there) are **quarantined** — form only. Our own curtailment claims would ground in the substrate (`get_plant.grid`, news CONSTRAINT lane) + named external sources.
