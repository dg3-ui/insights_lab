# Discussion — Reverse-Engineering an Outlier: the off-substrate method, with Red Dog Mine as the worked example

> **Status**: **OPEN**, opened 2026-06-14 — a **draft to argue with**, not a decision. Seeded by a 5-angle deep public-source research sweep on **Red Dog Mine** (Teck Resources / NANA, NW Arctic Alaska). This is the **companion mode to forward-deployment**: how an InfraSure analyst reasons about a climate-risk **outlier that is NOT in the MCP substrate** — and the reusable method that falls out of doing it once.
>
> **The trigger**: a promised internal bottom-up report on Red Dog (early next week), tied to the Teck "energy-built, modularly-applicable-to-mining" pilot (Friday 2026-06-12 meeting). **The deliverable we settled on**: *the reusable outlier method, with Red Dog as the worked proof — not a second Red Dog deck.*

## 1 · Two lanes — forward-deployment vs reverse-engineering

```text
FORWARD-DEPLOYMENT (everything so far)        REVERSE-ENGINEERING (this)
substrate-up: which (family × driver) cells   outlier-down: a specific off-substrate asset matters
  fit our data + our client book                (showcase / pilot), so show what the METHOD can do
grounded in the InfraSure MCP                  grounded in PUBLIC RESEARCH + the model + our resources
the moat = curated data + reasoning           the moat = the reasoning DISCIPLINE + a reusable method
```

**The banked insight (owner, 2026-06-14)** — this is a real architecture refinement, not a Red-Dog quirk:

```text
MCP            = TOP-DOWN / competitive grounding (rank a fleet, "you vs X across 15K plants")
RESOURCES + MODEL + RESEARCH = BOTTOM-UP grounding (a single asset's exposure)
→ on ANY bottom-up, the MCP is anyway limited; the strength comes from the model + the resource file.
  Red Dog is just the EXTREME case (zero MCP) — which makes it the cleanest demonstration of the bottom-up mode.
```

## 2 · The grounding stance (honest — read before building)

```text
GROUNDS ON     named public sources (research) + the model's current capability + our resource files
NOT            the public MCP · NOT model-gpr · NOT InfraSure proprietary data   (owner-confirmed)
FITS           the existing data_map: this is an `external`-DOMINANT resource. The discipline is UNCHANGED —
               every material claim → a named source + access date + confidence + caveat; blocked_claims still block.
```

**The A/B honesty.** A generic LLM + web could write *a* Red Dog climate narrative. So the moat here is **not data** — it's (a) the **grounding/gate discipline** (every claim sourced, dated, caveated; the gaps named; nothing over-quantified) and (b) the **reusable method**. If a Red Dog piece is presented to the team as "MCP-grounded InfraSure product," it would fail our own A/B and mislead. It must be labeled **research-grounded**.

## 4 · Red Dog evidence (the worked-example seed — sourced)

### The headline the deck missed — a coupled energy-logistics chokepoint (InfraSure's lane)

```text
Red Dog is OFF-GRID: ~25 MW typical demand on diesel gensets, ~15.5M gal/yr fuel, ~40,000 gal/day, barged in.
ONE 52-mi state-owned (AIDEA) gravel haul road → ONE shallow Chukchi-Sea port (no dock; lightered offshore),
ice-free only ~100 days/yr (Jul-Oct). That single window gates BOTH the year's diesel resupply (energy-IN)
AND the entire concentrate export (revenue-OUT).
→ Already realized: weather/sea conditions delayed shipments late 2025; Q4-2025 Red Dog gross profit fell to
  ~$200M (from ~$303M Q4-24) on the timing (Teck Q4-2025 release, Feb 2026).
```
This is the distinctive, defensible InfraSure framing — *energy-infrastructure-and-logistics under a changing ice/storm regime* — that the permafrost-only deck didn't make.

### The permafrost → water-balance crisis (primary-sourced)

```text
permafrost thaw → naturally elevated TDS in receiving waters → lost mixing-zone capacity → discharge blocked
+ a wetter Arctic → tailings/contact water +60% (3.3B→5.3B gal, 2017-21) → encroaches on dam freeboard
ADEC's own basis: stored water "threatens the safety of the tailings impoundment dams" (tail risk: dam failure).
Aug-2012 storm (15.2 in) EXCEEDED the tailings dam's design flood — the infra already ran outside its envelope.
Cost so far: ~$19M (2020); the fix under evaluation = a ~$279M / 50-mi pipeline to the Chukchi Sea (>half of
Red Dog's annual gross profit). Sources: ADEC APDES AK0038652 Statement of Basis (2026-04-09); Alaska Beacon /
Northern Journal (Dec 2025); Alaska Public Media (2020).
```

### The financial / ownership frame (why the adaptation decision is hard)

```text
A FADING asset (ore ~2032; output guided down to ~230-270K t by 2028) · a NANA royalty ratcheting 40%→45% (Oct
2027)→50% · ownership in flux (Anglo–Teck merger pending → a >70%-COPPER portfolio where a declining Arctic zinc
mine is non-core) · state closure assurance ~$558M + PERPETUAL post-closure water treatment (a liability that
outlives the revenue). → A $279M climate-adaptation capex collides with late-life economics — the real decision tension.
```

### The gaps ARE findings (a rigorous read names what it can't know)

```text
· days-of-fuel-AUTONOMY if an open-water window is missed/short — UNDISCLOSED (the latent single-point-of-failure)
· tailings-dam factor-of-safety / current freeboard margin — UNDISCLOSED
· foundation engineering of the fuel-tank farm (the Norilsk failure mode, §5) — UNDISCLOSED
· asset-level Red Dog P&L / adaptation capex — not broken out (reported inside Teck's zinc segment)
```

## 5 · The generalizable class (what makes the method reusable)

Red Dog instantiates a class, and the class is what the method captures:

```text
CLASS = remote, off-grid, cold-region / Arctic INDUSTRIAL asset, defined by two coupled traits:
   (1) electrically ISLANDED on a diesel/HFO microgrid, and
   (2) dependent on a SINGLE climate-sensitive seasonal logistics window to resupply fuel + export product.
HAZARD STACK: permafrost (foundation bearing-capacity loss + watershed chemistry) · a WETTER Arctic
   (precip/thaw overwhelming water containment) · coastal erosion · freeze-thaw · tundra wildfire.
```

Authoritative frameworks (citable, for the method's spine):
- **IPCC AR6 WG2 CCP6 (Polar)** — "warming and wetting"; most Arctic infrastructure impacted by mid-century (high confidence).
- **Hjort et al. 2018 (Nat. Comms)** — 33–69% of pan-Arctic infrastructure on thawing permafrost by 2050; and the sharp decision frame: **near-term risk is mitigation-INSENSITIVE → for an existing asset, ADAPTATION, not decarbonization, is the only lever.**
- **Rantanen 2022** — Arctic warming ~4× global (and models *under*-state it → the signal arrives faster, compressing adaptation lead-time).

Two precedents prove the abstraction generalizes:
- **Norilsk 2020** — a diesel storage tank on thaw-weakened permafrost piles collapsed → ~21,000 t spill, ~$2B damages. **The exact Red Dog risk geometry, realized.** (This is the single strongest proof the method matters — and it points straight at the undisclosed fuel-farm-foundation gap.)
- **Tibbitt–Contwoyto winter ice road** (NWT diamond mines) — the same narrowing-seasonal-window risk, *opposite sign* (needs ice **present**; Red Dog needs ice **absent**).

## 6 · The method outline (the reverse-engineering / outlier playbook — docs only)

A recipe-shaped, guided-reasoning spec (the model reads it; no engine):

```text
outlier_climate_exposure  (reverse-engineering playbook)
1. CLASSIFY     is the asset off-substrate? what class (Arctic-industrial / remote-offgrid / coastal / …)?
2. ASSEMBLE     build the exposure profile asset-by-asset from PUBLIC sources (there's no registry) —
                power source, logistics window, foundation/permafrost, water, the hazard stack
3. APPLY        the class hazard taxonomy + the citable frameworks (IPCC CCP6 / Hjort / AMAP) + a precedent (Norilsk)
4. GROUND       every claim → named source + date + confidence + fact-vs-projection label
5. NAME GAPS    the undisclosed asset-level numbers ARE findings (days-of-autonomy, dam FoS, foundation eng.)
6. FRAME        the decision (for an existing asset: the ADAPTATION lever + the late-life-economics tension)
7. GATE         block what we can't model — no $ EAL, no return-periods, no plant-level forecast, no single-cause
8. RENDER       (post-gate) the team artifact / pilot support
```

It composes: a **new `external`-grounded outlier resource** (the class method) + the existing **`_method` primitives** (screen/rank/score · non-stationarity · EVT) + the **cloud skill** (deep public research) + the **model**. The public MCP plays **only** a bit part (a competitive/peer overlay if any in-substrate asset is comparable — for Red Dog, ~none).

```text
   core substrate-grounded lab. Reusable method = the guardrail against "write me any climate report" drift.
2. HONESTY — research-grounded, not MCP/model-gpr. Never present it as substrate-grounded product.
3. DON'T RE-DO THE DECK — additive (current · energy-angle · generalizable · gap-honest) or don't ship it.
4. DON'T OVER-QUANTIFY — no $ EAL/PML, no return-periods, no single-cause attribution; the model-gpr-class numbers
   stay blocked exactly as in the substrate resources. Gaps are findings, not things to fill with a guess.
5. ONE ASSET / ONE CLASS per worked example; the discipline (claim grammar, gate) is identical to a substrate resource.
```
